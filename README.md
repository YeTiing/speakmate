# FluentLoop — AI 英语口语教练

基于场景驱动的英语口语练习工具。通过模拟真实对话场景，帮助学习者练习口语、获得纠错反馈，并在每次练习后生成可量化的评分报告。

## 功能特性

- **内置场景**：求职面试、餐厅点餐、商务会议、机场值机、酒店入住、日常闲聊
- **自定义场景**：手动创建场景，或用中文 prompt 一键 AI 生成
- **语音工作流**：浏览器语音识别 → 转写确认/编辑 → AI 角色回复 → 回放 → 语音合成播报
- **文字兜底**：可直接打字输入英文，或输入中文自动翻译成英文再发送
- **AI 对话**：通过本地 API 代理调用 LLM。未配置 key 时自动降级为确定性本地对话
- **结构化结束**：AI 可标记对话为 "已完成" 或 "已终止"，自动跳转评分报告
- **发音评估**：可选接入 Azure 发音评估。未配置时使用启发式评分并标注置信度
- **自适应纠错**：阻塞性错误在对话中即时纠正，次要问题汇总到报告中
- **评分报告**：综合评分、发音/流利度/语法/词汇/互动/场景完成度、纠错记录、评分依据、置信度说明、下次练习建议
- **输入感知评分**：语音和文字回答分开统计，发音和流利度仅基于语音作答
- **本地历史**：localStorage 存储近期会话、分数趋势和历史报告

## 产品流程

```
选择或生成场景
  → 开始练习
  → AI 角色主动开场
  → 学习者语音或打字回答
  → 语音转写确认/编辑后发送
  → AI 结合上下文做出角色回应
  → 可翻译消息或重播音频
  → 对话正常结束或终止
  → 生成含评分依据和置信度说明的报告
  → 本地保存会话历史
```

## 架构

```
server/
  index.mjs        本地 API 代理，转发 LLM/翻译/场景生成/发音评估请求
src/               （前端源码，需要先 npm run build）
```

浏览器 SpeechRecognition / SpeechSynthesis 处理语音输入输出，localStorage 做持久化。

## 快速开始

确保已安装 Node.js 20+。

```bash
npm install
npm start
```

**启动后打开浏览器访问 `http://127.0.0.1:8787`**（仅限本机访问，默认不对外暴露）。

开发模式（支持前端热更新）：

```bash
npm run dev
```

## 环境变量配置

复制 `.env.example` 为 `.env` 并填入 key：

| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | DeepSeek API 密钥（推荐） |
| `OPENAI_API_KEY` | OpenAI API 密钥（备选） |
| `AZURE_SPEECH_KEY` | Azure 语音服务密钥 |
| `AZURE_SPEECH_REGION` | Azure 语音服务区域 |

配置 LLM key 后可使用 AI 对话、翻译和场景生成。配置 Azure 后可获得专业发音评估。

## 评分模型

| 维度 | 权重 |
|------|------|
| 发音 | 25% |
| 流利度 | 20% |
| 语法 | 20% |
| 词汇 | 15% |
| 互动 | 10% |
| 场景完成度 | 10% |

报告包含：场景目标完成情况、语音/文字作答数量、语速、停顿分析、填充词、平均回答长度、发音评估模式、AI 响应延迟、语法/表达纠错、评分置信度说明。

## 浏览器兼容

语音识别依赖浏览器 `SpeechRecognition` / `webkitSpeechRecognition` 支持。Chrome 系浏览器效果最佳。语音识别不可用或不准确时，可使用文字输入兜底。

## 纠错策略

- **阻塞性错误**：影响意思理解时，对话中即时纠正
- **非阻塞问题**：语法/表达小问题，汇总到报告中
- **严格模式**：适合需要密集纠错的学习者
